- **범용 (General Purpose)**: 균형 잡힌 CPU, 메모리, 스토리지. 웹 서버나 앱에 적합.
  - 예: t3.micro (무료 티어), m5.large, m6g.medium.

- **컴퓨팅 최적화 (Compute Optimized)**: 고성능 CPU. 배치 처리나 과학 계산에 좋음.
  - 예: c5.large, c6i.xlarge.

- **메모리 최적화 (Memory Optimized)**: 대용량 메모리. 데이터베이스나 빅데이터에 적합.
  - 예: r5.large, x1e.32xlarge (최대 메모리).

- **스토리지 최적화 (Storage Optimized)**: 고속 스토리지. 빅데이터나 파일 서버에 좋음.
  - 예: i3.large (NVMe SSD), d2.xlarge (HDD).

- **가속 컴퓨팅 (Accelerated Computing)**: GPU/FPGA. 머신러닝이나 그래픽스에 특화.
  - 예: p3.2xlarge (NVIDIA GPU), g4dn.xlarge.

- **고성능 컴퓨팅 (High Performance Computing)**: 클러스터링 지원. HPC 워크로드.
  - 예: hpc6a.48xlarge.



## EC2 instance-type


### AWS EC2 인스턴스 타입 개요
이 리스트는 Amazon Web Services (AWS) Elastic Compute Cloud (EC2) 인스턴스 타입으로, 주로 가변성 워크로드(예: 웹 서버, 개발 환경)에 적합한 "버스트 가능(Burstable)" 인스턴스들입니다. CPU 성능이 필요할 때 크레딧을 사용해 일시적으로 부스트되며, 기본적으로 저비용으로 설계되었습니다. 주요 차이점은 **세대(Generation)**, **CPU 아키텍처**, **크기(Size)**에 따라 나뉩니다. 아래에서 자세히 설명하겠습니다.

### 세대 및 아키텍처 차이
- **t2 시리즈**: 가장 오래된 세대(2014년 출시). Intel Xeon 프로세서를 사용하며, 기본 성능이 낮아 크레딧에 의존하는 경우가 많음. 비용 효율적이지만, 최신 세대에 비해 느림.
- **t3 시리즈**: t2의 후속(2018년 출시). Intel Xeon 기반으로, t2보다 더 나은 CPU 성능과 메모리 효율을 제공. 크레딧 시스템은 유사하지만, 더 빠른 버스트 가능.
- **t3a 시리즈**: t3의 AMD 버전(2019년 출시). AMD EPYC 프로세서를 사용해 t3보다 저비용(약 10% 절감)으로 비슷한 성능을 냄. Intel 기반 t3에 비해 전력 효율이 좋음.
- **t4g 시리즈**: 최신 세대(2020년 출시). AWS Graviton2 프로세서(ARM64 아키텍처)를 사용해 t3보다 40% 더 나은 가격 대비 성능. ARM 기반이라 Intel/AMD와 호환되지 않는 소프트웨어는 추가 설정 필요. 저비용, 고효율이 장점.

모든 시리즈는 버스트 가능 인스턴스로, CPU 크레딧을 모아 사용합니다. 크레딧이 부족하면 기본 성능으로 떨어짐.

### 크기별 스펙 비교
크기는 nano(가장 작음)부터 2xlarge(가장 큼)까지, vCPU(가상 CPU 코어)와 메모리(RAM)가 증가합니다. 아래 표는 각 크기의 일반적인 스펙(모든 세대에 공통 적용, 약간의 변동 가능)을 요약한 것입니다. 실제 스펙은 AWS 문서에서 확인하세요.

| 크기       | vCPU | 메모리 (GiB) | 네트워크 대역폭 (Gbps) | EBS 대역폭 (Mbps) | 설명 |
|------------|------|--------------|-------------------------|-------------------|------|
| nano      | 1    | 0.5          | 최대 5 (버스트)        | 최대 2,085        | 초소형, 테스트나 간단한 앱에 적합. |
| micro     | 1    | 1            | 최대 5 (버스트)        | 최대 2,085        | 기본적인 워크로드, 예: 작은 웹사이트. |
| small     | 1    | 2            | 최대 5 (버스트)        | 최대 2,085        | micro보다 메모리 많음, 가벼운 앱. |
| medium    | 2    | 4            | 최대 5 (버스트)        | 최대 2,085        | 다중 스레드 앱이나 중간 규모 워크로드. |
| large     | 2    | 8            | 최대 5 (버스트)        | 최대 2,780        | 더 많은 메모리, 데이터베이스나 캐싱에 유용. |
| xlarge    | 4    | 16           | 최대 5 (버스트)        | 최대 2,780        | 고부하 앱, 병렬 처리. |
| 2xlarge   | 8    | 32           | 최대 5 (버스트)        | 최대 2,780        | 대규모 워크로드, 고성능 필요 시. |



## EBS

### EBS란?
EBS(Elastic Block Store)는 Amazon Web Services(AWS)의 블록 스토리지 서비스로, EC2 인스턴스에 연결할 수 있는 영구적인 스토리지입니다. "탄력적 블록 저장소"로 번역되며, 하드 드라이브처럼 데이터를 저장하고 읽을 수 있습니다. EC2 인스턴스가 종료되더라도 데이터를 유지할 수 있어, 데이터베이스나 파일 저장에 유용합니다.

### 주요 특징
- **영구성**: 인스턴스가 중지되거나 재부팅되어도 데이터가 사라지지 않음 (단, 인스턴스 삭제 시 옵션에 따라 유지 가능).
- **볼륨 타입**: 성능에 따라 여러 종류.
  - **gp3/gp2**: 범용, 저비용, 일반 앱에 적합 (gp3는 더 저렴하고 빠름).
  - **io2/io1**: 고성능, IOPS(초당 입출력 연산)가 높아 데이터베이스에 좋음.
  - **st1/sc1**: 저비용, 대용량 데이터 처리에 최적.
- **스냅샷**: 백업 기능으로, 볼륨의 데이터를 이미지로 저장해 복원 가능.
- **확장성**: 용량을 동적으로 늘릴 수 있음 (예: 1GB에서 수 TB까지).
- **가격**: 사용한 스토리지와 IOPS에 따라 요금 부과 (월별 GB 단위).

### EC2와의 관계
EC2 인스턴스에 EBS 볼륨을 연결하면, 인스턴스의 루트 디스크(부팅용)나 추가 스토리지로 사용할 수 있습니다. 예를 들어, t3.micro 인스턴스에 EBS를 붙여 웹사이트 데이터를 저장할 수 있음. 이전 응답에서 언급한 "EBS 대역폭"은 인스턴스가 EBS에 데이터를 읽고 쓰는 속도를 의미합니다.
