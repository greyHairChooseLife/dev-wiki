# Next.js Static Assets (Public Directory)

`public` 디렉토리에 있는 파일은 `/` 경로로 바로 접근할 수 있습니다.

## 1. 디렉토리 구조

```text
public/
└── logo.png
app/
└── page.tsx
```


## 2. 사용 방법

```tsx
// app/page.tsx
import Image from 'next/image';

export default function Page() {
  return (
    <div>
      {/* 1. 일반 img 태그 */}
      <img src="/logo.png" alt="Logo" />

      {/* 2. Next.js Image 컴포넌트 (권장) */}
      <Image 
        src="/logo.png" 
        alt="Logo" 
        width={100} 
        height={100} 
      />
    </div>
  )
}
```



## 3. Image 컴포넌트 권장 이유

Next.js `<Image>` 컴포넌트는 **성능 최적화를** 자동으로 처리합니다.

| 기능 | `<img>` 태그 | `<Image>` 컴포넌트 |
| :--- | :--- | :--- |
| **이미지 포맷** | 원본 그대로 전송 | WebP/AVIF 자동 변환 (용량 감소) |
| **사이즈 최적화** | 원본 크기 전송 | 디바이스에 맞는 크기로 자동 리사이징 |
| **로딩 방식** | 즉시 로드 | Lazy Loading (화면에 보일 때 로드) |
| **레이아웃** | 로딩 중 화면 밀림 발생 | 공간을 미리 확보해 밀림 방지 (CLS 개선) |
