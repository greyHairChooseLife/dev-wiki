

리액트(React)와 리액트 라우터(React Router)의 관계는 **싱글 페이지 애플리케이션(SPA)**의 작동 방식과 밀접하게 연관되어 있습니다.

### 리액트와 SPA의 특성

리액트는 기본적으로 **UI 라이브러리**입니다. 리액트로 만든 웹 애플리케이션은 보통 하나의 HTML 파일(`index.html`)만 로드하고, 자바스크립트를 통해 화면의 내용을 동적으로 바꿔치기하는 SPA 방식을 사용합니다.

전통적인 웹사이트(Multi Page Application)는 사용자가 링크를 클릭할 때마다 서버에 새로운 HTML 파일을 요청하고 페이지 전체를 새로고침합니다. 반면, SPA는 처음에 모든 리소스를 받거나 필요한 데이터만 비동기로 받아와 화면의 일부만 갱신하므로 사용자 경험이 훨씬 부드럽습니다.

### 왜 리액트 라우터가 필요한가?

리액트 자체에는 **"주소(URL)에 따라 다른 화면을 보여주는 기능"**이 내장되어 있지 않습니다. 단순히 컴포넌트를 렌더링할 뿐입니다. 여기서 리액트 라우터가 필요해집니다.

1.  **URL과 UI의 동기화:** 사용자가 `/about` 주소로 들어왔을 때 `About` 컴포넌트를 보여주고, `/contact`로 들어왔을 때 `Contact` 컴포넌트를 보여주는 역할을 합니다.
2.  **새로고침 없는 페이지 이동:** 브라우저의 주소창은 바뀌지만, 실제로 서버에 새 페이지를 요청하지 않고 자바스크립트로 화면만 교체합니다. (HTML5 History API 사용)
3.  **뒤로 가기/앞으로 가기 지원:** 브라우저의 히스토리를 관리하여 사용자가 뒤로 가기 버튼을 눌렀을 때 이전 컴포넌트로 정상적으로 돌아가게 해줍니다.
