
3주라는 짧은 기간 동안 실전 프로젝트를 완수해야 한다면, 이론적인 깊이보다는 **"기능 구현을 위한 필수 도구"** 위주로 학습 범위를 좁혀야 합니다. 중복을 제거하고, **실제 개발 순서**에 맞춘 로드맵을 제안합니다.


**추천 학습 전략:**
1.  **1단계(UI)**로 메인 페이지 껍데기를 먼저 만듭니다.
2.  **2단계(State)**로 버튼을 눌렀을 때 숫자가 바뀌거나 모달이 뜨게 만듭니다.
3.  **3단계(Effect)**로 가짜 데이터(JSON)를 불러와서 리스트에 뿌려봅니다.


### **[1단계: 화면 그리기 (UI 구성)] - 1~2일**

가장 먼저 정적인 화면을 컴포넌트로 쪼개서 그리는 법을 익힙니다.

1.  **JSX 문법 (HTML과 다른 점만 체크)**
    *   `class` 대신 `className`
    *   변수 넣을 땐 `{ }` (중괄호)
    *   닫는 태그 필수 (`<input />`)
2.  **Props (데이터 전달)**
    *   부모가 자식에게 데이터(변수, 함수)를 넘겨주는 방법
    *   *핵심:* "데이터는 위에서 아래로 흐른다"
3.  **조건부 & 반복 렌더링 (필수)**
    *   `map()`: 배열 데이터를 리스트로 뿌리기 (for문 대신 사용)
    *   삼항 연산자 (`condition ? A : B`): if문 대신 사용



### **[2단계: 살아있는 화면 만들기 (상태 관리)] - 3~5일**

사용자의 입력에 반응하고 화면을 갱신하는 핵심 단계입니다. 여기가 제일 중요합니다.

1.  **useState (상태 선언)**
    *   변수 대신 `useState`를 써야 화면이 바뀐다는 점 이해
    *   `const [값, 변경함수] = useState(초기값)`
2.  **이벤트 핸들링**
    *   `onClick`, `onChange` 연결하기
    *   입력값(`input`)을 state로 관리하기
3.  **State 끌어올리기 (Lifting State Up)**
    *   *가장 중요한 개념:* 형제 컴포넌트끼리 데이터를 주고받으려면, **부모 컴포넌트에 state를 만들고 props로 내려줘야 함**을 이해



### **[3단계: 외부와 소통하기 (서버 통신)] - 2~3일**

백엔드 API에서 데이터를 가져오거나 보낼 때 필요합니다.

1.  **useEffect (부수 효과)**
    *   "화면이 처음 떴을 때" 서버에서 데이터 가져오기
    *   의존성 배열(`[]`)의 역할 이해 (언제 실행될지 결정)
2.  **비동기 통신 (fetch / axios)**
    *   API 호출해서 받아온 데이터를 `useState`에 넣기



### **[4단계: 프로젝트 구조 잡기 (조립)] - 나머지 기간**

여러 페이지를 만들고 앱을 완성하는 단계입니다.

1.  **React Router**
    *   페이지 이동 구현 (`<Link>`, `useNavigate`)
2.  **폴더 구조**
    *   `components` (재사용 버튼, 입력창) vs `pages` (전체 화면) 분리
3.  **전역 상태 (선택 사항)**
    *   *주의:* 처음부터 Redux 같은 거 쓰지 마세요.
    *   Props로 넘겨주기 너무 귀찮을 때만 `Context API`나 `Zustand` 같은 가벼운 것 고려.
